# Сравнение стоимости квартир (Красноярск, Avito)

Веб-приложение на Flask для сравнения цены конкретной квартиры с
аналогами в том же районе/микрорайоне Красноярска. Аналоги собираются
с Avito в реальном времени с помощью Playwright + BeautifulSoup,
геокодируются через OpenStreetMap (Nominatim) и фильтруются по
расстоянию (по умолчанию ≤ 2 км).

> **Образовательный проект.** Использование парсера должно
> соответствовать пользовательскому соглашению Avito и применяться в
> образовательных целях. Авторы не несут ответственности за
> недобросовестное использование.

---

## Возможности

- Поисковая форма с обязательным выбором количества комнат
  (студия / 1 / 2 / 3 / 4+) и опциональными параметрами (площади,
  этаж, год постройки, цена).
- Дополнительные фильтры: радиус (по умолчанию 2 км, шаг 0.5 км),
  допустимое отклонение площади (±15%), глубина поиска (5–20).
- Жёсткая фильтрация по числу комнат на этапе сбора (ставится URL
  Avito с нужным сегментом: `studii`, `2-komnatnye` и т.п.).
- Автоматическая геовалидация каждого аналога (Nominatim + geopy),
  кеширование координат в SQLite.
- Анти-бан меры парсера: stealth, рандом задержек 2–5 / 10–20 сек,
  ротация User-Agent и viewport, поддержка пользовательских cookies
  (`avito_cookies.json`).
- Аналитика: средняя/медианная цена за м², средний год постройки,
  отклонения, предупреждения (цена завышена, дом значительно
  старше/новее аналогов и т.п.).
- Гистограмма распределения цен на Chart.js.
- Сортируемая таблица аналогов с прямыми ссылками.
- Экспорт в CSV (Excel-совместимый, разделитель `;`).
- История последних 10 запросов в SQLite.
- Шеринг результата по уникальной ссылке (24 часа).
- Прогресс-бар парсинга по WebSocket (Flask-SocketIO) + fallback на
  HTTP-поллинг.

## Быстрый старт (для свежеустановленного Python)

### Вариант A. Локально

1. Установите **Python 3.11** или новее.
   * Windows: <https://www.python.org/downloads/>
   * macOS: `brew install python@3.11`
   * Ubuntu: `sudo apt install python3.11 python3.11-venv`

2. Клонируйте проект и зайдите в каталог:
   ```bash
   git clone <url-репозитория>
   cd Indicidualproject
   ```

3. Создайте виртуальное окружение и активируйте его:
   ```bash
   python3.11 -m venv .venv
   # Linux/macOS
   source .venv/bin/activate
   # Windows (PowerShell)
   .venv\Scripts\Activate.ps1
   ```

4. Установите зависимости и Chromium для Playwright:
   ```bash
   pip install --upgrade pip
   pip install -r requirements.txt
   playwright install chromium
   ```

5. Запустите приложение:
   ```bash
   python app.py
   ```
   Откройте в браузере: <http://127.0.0.1:5000>.

### Вариант B. Docker

```bash
docker compose up --build
```
Браузер: <http://127.0.0.1:5000>. Образ
`mcr.microsoft.com/playwright/python` уже содержит Chromium со всеми
зависимостями — `playwright install` запускать не нужно.

## Конфигурация

Все настройки — в `config.py`:

| Параметр | Назначение | Дефолт |
|----------|------------|--------|
| `MAX_PAGES` | Сколько страниц поиска листать | `5` |
| `DEFAULT_RESULTS` | Сколько строгих аналогов искать | `10` |
| `MAX_RESULTS_HARD_CAP` | Абсолютный максимум | `20` |
| `DELAY_ACTION_MIN/MAX` | Задержки между мелкими действиями | `2–5 сек` |
| `DELAY_PAGE_MIN/MAX` | Задержки между страницами поиска | `10–20 сек` |
| `DEFAULT_DISTANCE_KM` | Радиус поиска по умолчанию | `2.0` |
| `MIN_STRICT_MATCHES` | Минимум строгих совпадений по комнатам | `5` |
| `SHARE_TTL_HOURS` | Срок жизни шаринг-ссылки | `24` |

Переопределить порт/хост/секрет можно через переменные окружения:
```bash
export FLASK_SECRET_KEY=$(python -c "import secrets;print(secrets.token_hex(32))")
export PORT=5000
export HOST=0.0.0.0
```

## Реальные cookies Avito (опционально)

Чтобы снизить вероятность блокировки, можно подложить экспортированные
cookies из реального браузера:

1. Откройте Avito в Chrome, авторизуйтесь.
2. Установите расширение «Cookie-Editor», экспортируйте cookies в JSON.
3. Сохраните файл рядом с `app.py` под именем `avito_cookies.json`
   (он добавлен в `.gitignore`).

Парсер автоматически подхватит cookies при старте.

## Известные ограничения

- Avito регулярно меняет вёрстку и активно борется с автоматизацией.
  Для стабильной работы в продакшене потребуются прокси и регулярная
  поддержка селекторов.
- Nominatim ограничивает 1 запрос/сек на пользователя — приложение
  это уважает (см. `geocoder.py`). Все ответы кешируются.
- Кеш ответов парсинга в БД хранит только метрики — повторный поиск
  выполняет реальный запрос к Avito.
